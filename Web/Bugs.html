Known Bugs

Bugs in the typesetter

  • Subscripts accented mathematical characters on wrong place.
  • When entering a formula like sqrt(frac(1+sqrt(frac(1,2)),..) in math mode, the square roots do not have the right size.
  • Big delimiters are not properly taken into account inside macros.
  • The inter-paragraph space between paragraph 1 and paragraph 2 should be determined otherwise (see programming session in letter style) as the current environment inter-paragraph space at the end of paragraph 1.
  • left-. and right-. do not work.
  • ``By'' is not translate in article style.


Other bugs

  • Problems with X colormaps on 24 bits and 32 bits displays.
  • The user should be prompted for unsaved files when closing TeXmacs from outside TeXmacs.
  • Quickly crash TeXmacs by typing esc s enter backspace.
  • Alternative way: enter math-mode; type minus; select font size ``very large''; type minus.
  • When deleting a theorem, the numbers of the following theorems are not updated.
  • When running TeXmacs inside WindowMaker, it doesn't have the dockable icon.


Problems when compiling with egcs

Unfortunately, several bugs seem to persist in the egcs compiler. In order to compile TeXmacs using egcs 2.95.2, optimization and redhat linux, I had to change correct source at three places in order to avoid segmentation faults. On a SuSE linux system, I even had to change one line in order to get the non optimized program working, and I still don't manage to correctly compile the program using optimization.

A typical example of the kind of changes needed to get things working is avoiding nested expressions : 

    SI
    TER::get_length (string name) {
      string s= get (name)->label;
      return decode_length (s);
    }
instead of 
    SI
    TER::get_length (string name) {
      return decode_length (get (name)->label);
    }
